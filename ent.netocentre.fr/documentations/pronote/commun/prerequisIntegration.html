Informer le support

Lors d'une première intégration de pronote dans l'ENT, il est nécessaire que le support en soit averti, surtout dans le cas d'un Pronote installé dans l'établissement pour lequel la DSI du rectorat doit déployer des services afin de le rendre disponible dans l'ENT. Cela est aussi nécessaire afin de fournir des accès sécurisés aux services Pronote.
Lors du passage de Pronote en mode hébergé chez Index Éducation, ou inversement un retour à l'installation dans l'établissement, il est également nécessaire d'informer le support afin de changer le lien d'accès à Pronote configuré dans l'ENT.

Normalisation des installations

Il est fortement recommandé de faire héberger son serveur Pronote chez Index Éducation car cela facilite l'administration.
Mais s'il est choisi d'héberger Pronote en établissement, afin de savoir si l'installation respecte les normes il est important de contacter le support de la DSI du rectorat.
Pour des raisons évidentes de sécurité, le système d'exploitation de type Windows où les services Pronotes sont installés doit être un système de type serveur, mis à jour régulièrement (màj automatiques) et dont le support Microsoft est encore valable (windows XP à bannir, par exemple). Suite à cette installation les services Pronote seront exposés sur internet.

Création de la base pronote

Avant de configurer Pronote pour son intégration dans l'ENT, il faut que celui-ci ait une base initialisée correctement à partir des informations de STS-web et de Siècle. Mais aussi, ces applications doivent (facultatif mais vivement recommandé) au préalable avoir été complétées par les informations EDT. Les saisies manuelles de compte pour initialiser la base pronote sont à banir, les procédures automatiques évitent ces saisies et facilitent le rapprochement des comptes ENT/pronote par la suite.
Ces opérations sont recommandées dans le but de faciliter la gestion du rapprochement des comptes ENT avec les comptes Pronote dans la base pronote par les établissements.

Accessibilité

Une fois intégrés dans l'ENT les services Pronote doivent être fonctionnel en permanence, cela signifie que :

  • La base pronote est publiée
  • Les espaces des utilisateurs autorisés sont publiés
  • Les applications sont configurées comme des services Windows afin qu'elles soient automatiquement démarrés au redémarrage de la machine et/ou du système.
En résumé sans un accès pronote.net fonctionnel l'accès à pronote dans l'ENT ne sera pas possible, car tout dépend du fonctionnement du service pronote.net .
